# Backlog - User Stories

This folder contains user stories created by the Product Owner agent.

## Purpose

User stories document WHAT needs to be built from the user's perspective, not HOW to build it (that's the Architect's job).

## User Story Template

```markdown
# User Story: [Short Title]

**As a** [type of user]
**I want** [goal/desire]
**So that** [benefit/value]

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Priority

[High / Medium / Low]

## Status

[New / Ready / In Progress / Done]

## Notes

[Any additional context or clarifications]
```

## Naming Convention

`YYYY-MM-DD-short-description.md`

Example: `2026-01-30-joke-website.md`
